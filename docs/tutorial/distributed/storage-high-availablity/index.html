数据如何复制

主备复制

最简单的复制方案就是主备复制，MySQL、Redis、MongoDB 都提供了这种复制方案。

缺点：故障的时候，需要人工干预，硬件上有浪费。适合于内部的后台管理系统。

主从复制

从意味着是要干活的，也就是从节点提供读的能力，主提供了读写能力。

缺点：故障的时候，需要人工干预。一般适用于写少读多的业务。

主备倒换/主从倒换

主主复制

因为数据必须满足可以双向复制，因此适合临时性、可丢失、可覆盖的数据场景
